# res-downloader

[GitHub地址](https://github.com/putyy/res-downloader)

## 项目简介

`res-downloader` 是一款轻量级的浏览器扩展（兼容 Chrome/Edge/Firefox），可帮助用户一键下载当前网页中所有资源（图片、音频、视频、JavaScript、CSS 等），并支持按资源类型过滤、批量下载、保存为 ZIP 包等功能。它简化了从大型网站或单页应用中提取多媒体素材的过程，适用于 web 开发者、内容采集人员以及需要离线缓存的普通用户。

## 主要特性

| 功能 | 说明 |
|------|------|
| **一键下载** | 直接点击扩展图标，弹出下载面板；插件自动扫描当前页并列出可下载的资源。 |
| **资源类型过滤** | 支持通过下拉筛选框选择仅下载图片、音频、视频、脚本或样式表等。 |
| **自定义下载路径** | 允许在下载前选择本地磁盘目录（通过系统对话框）。 |
| **ZIP 导出** | 自动将多文件打包为 ZIP，方便后续上传或共享。 |
| **预览功能** | 资源列表中可预览图片、视频、音频；JS/CSS 通过内置编辑器查看源代码。 |
| **批量操作** | 支持全选/反选、单选选择后统一下载、或按文件类型批量操作。 |
| **支持跨域资源** | 通过 `proxy` 或 `CORS` 解决跨域限制，极大提升兼容性。 |
| **可视化状态指示** | 下载进度条、成功/失败提示，避免用户误操作。 |
| **配置存储** | 所有设置保存在浏览器本地存储，插件升级后保持不变。 |
| **简化 API** | 公开 100% 清洁、易用的官方 API 供开发者快速集成。 |

## 安装使用

### 1. 安装插件

1. 打开 Chrome 浏览器，前往  
   `chrome://extensions/` 并开启 **开发者模式**。  
2. 点击 **加载已解压的扩展程序**，选择本仓库根目录（`res-downloader` 目录）。  
3. 完成后图标会显示在浏览器工具栏。  
> 对于 Edge/Firefox，直接下载 `.xpi/.crx` 文件后拖拽至扩展页面即可。

> 🔌 **注意**：首次使用时，浏览器可能会弹出安全提示，需允许插件访问页面内容。

### 2. 开始下载

1. 打开任意网页（支持普通网页、单页应用、动画等）。  
2. 单击工具栏上的 `res-downloader` 图标，弹出下载面板。  
3. 通过 **筛选** 下拉框选择需要的资源类型。  
4. 对列表进行 **勾选/取消勾选**：  
   - `全选`、`反选` 快捷按钮。  
   - 单击文件名可预览（图片/视频/音频）或弹出代码面板（JS/CSS）。  
5. 点击 `下载` 或 `下载为 ZIP` 按钮。  
6. 选择下载目录（默认在系统下载文件夹）。完成后即可离线访问资源。

### 3. 配置项（可选）

在插件页面（右键图标 → `选项`），可以做以下自定义：

| 配置 | 默认 | 作用 |
|------|------|------|
| `自动过滤重复路径` | `开` | 是否跳过已下载的同路径文件。 |
| `默认下载目录` | 浏览器下载文件夹 | 更改下载后文件夹位置。 |
| `启用跨域代理` | `关` | 自动使用代理解决 CORS 问题。 |
| `按文件类型打包` | `关` | 将相同类型的文件单独打包为不同 ZIP。 |

### 4. 开发者使用

```javascript
// 只需 import core module
import { downloadResources } from 'res-downloader';

// example: 通过 JS 手动触发下载
downloadResources({
  url: window.location.href,  // 当前页
  types: ['image', 'audio'],  // 只下载图片与音频
  onProgress: (info) => console.log(info),
});
```

> 该 API 提供了完整的错误处理和进度回调，方便在自己的项目中集成。

## 常见问题

| 问题 | 解决方案 |
|------|----------|
| 资源下载失败 | 检查页面是否开启了 CSP 及 CORS；尝试开启 `跨域代理`；确认资源链接是否完整。 |
| 预览图片不显示 | 资源链接可能是 `data:` 或 `blob:`，可先下载后再手动查看。 |
| 向服务器频繁请求导致被限流 | 使用 `下载为 ZIP` 模式，一次性打包可大幅降低请求次数。 |
| 插件无法识别脚本资源 | 有些脚本为动态生成（`<script type="module">`），需在页面条件下手动刷新后再扫描。 |
| 下载的文件名乱码 | Windows 或 macOS 可能对文件名字符有限制，插件已进行 URL 解码。 |
| `res-downloader` 与其他下载器冲突 | 确认浏览器已将其它下载器禁用或做 URL 优先级处理。 |
