## MineContext：用上下文创造，从混乱中带来清晰

MineContext 是一个开源的主动上下文感知 AI 伙伴，致力于为您的学习、工作和创造带来清晰和效率。它通过截图和内容理解（未来支持多源多模态信息，包括文档、图像、视频、代码和外部应用数据），能够看到并理解用户的数字世界上下文。基于底层上下文工程框架，它主动提供高质量信息，如见解、每日/每周总结、待办事项和活动记录。

### 主要功能

MineContext 专注于四个关键功能：无忧收集、智能重现、主动交付和上下文工程架构。

1. **📥 无忧收集**：能够收集和处理海量上下文。设计的存储管理使广泛收集不会增加心理负担。
2. **🚀 主动交付**：在日常使用中主动交付关键信息和见解。它从您的上下文中提取总结内容，如每日/每周总结、提示和待办事项，并直接推送到您的主页。
3. **💡 智能重现**：在创造过程中智能地重现相关和有用的上下文。确保辅助创造而不让信息淹没您。
4. **🎯 上下文工程架构**：支持多模态、多源数据的完整生命周期——从捕获、处理和存储到管理、检索和消费——从而生成六种类型的智能上下文。

### 隐私保护

MineContext 高度重视用户隐私。默认情况下，所有数据都存储在本地路径 `~/Library/Application Support/MineContext/Data` 中，以确保您的隐私和安全。此外，我们支持基于 OpenAI API 协议的自定义模型服务。您可以在 MineContext 中使用完全本地的模型，确保任何数据都不会离开您的本地环境。

### 快速开始

1. **安装**：从 [GitHub 最新发布](https://github.com/volcengine/MineContext/releases) 下载应用程序。

2. **禁用隔离属性**：在终端输入以下命令以禁用隔离属性，然后运行应用程序。

   ```
   sudo xattr -d com.apple.quarantine "/Applications/MineContext.app"
   ```

3. **输入 API 密钥**：应用程序启动后，按照提示输入您的 API 密钥。我们支持豆包、OpenAI 和自定义模型服务。推荐使用豆包模型，API 密钥可在 [API 管理界面](https://console.volcengine.com/ark/region:ark+cn-beijing/apiKey) 生成。

4. **开始录制**：进入 [屏幕监控] 以启用屏幕共享系统权限。设置完成后，重启应用程序。然后在 [设置] 中设置屏幕共享区域，点击 [开始录制] 以开始截图。

5. **忘记它**：开始录制后，您的上下文将逐渐被收集。它需要一些时间来生成价值。所以，放心忘记它，专注于其他任务。MineContext 将在后台为您生成待办事项、提示、总结和活动。当然，您也可以通过 [与 AI 聊天] 进行主动问答。

6. **后端调试**：MineContext 支持后端调试，可在 `http://localhost:1733` 访问。

### 目标用户

- **知识工作者**：研究人员、分析师——导航大量信息，提高信息处理和分析效率。
- **内容创作者**：作家、博主——渴望无尽灵感，优化内容创作工作流程。
- **终身学习者**：学生、研究人员——构建系统知识体系，高效管理和连接学习材料。
- **项目管理者**：产品经理、项目经理——整合多源信息和数据，确保项目一致性和决策效率。

### 上下文来源

MineContext 支持多种上下文来源，包括屏幕截图、笔记编辑、链接上传、文件上传等，未来将扩展到更多来源如会议记录、RSS、AI 对话等。
