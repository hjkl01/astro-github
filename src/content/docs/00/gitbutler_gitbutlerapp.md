# GitButler（gitbutlerapp/gitbutler）

**项目地址**  
<https://github.com/gitbutlerapp/gitbutler>

---

## 概述  
GitButler 是一个跨平台的 Git GUI 客户端，旨在帮助开发者更高效地管理代码仓库、分支与 Pull Request。它集成了 GitHub、GitLab、Bitbucket 等代码托管服务，支持在本地与远程仓库间无缝切换，提供直观的界面与丰富的工作流工具。

---

## 主要特性  

| 特性 | 描述 |
|------|------|
| **分支管理** | 通过可视化分支图表快速定位、切换、合并、删除分支。支持命令行与图形化操作。 |
| **Pull Request（PR）一站式处理** | 直接在客户端创建、查看、评论、合并 PR；同步 PR 状态与远程仓库。 |
| **分支从 Issue 自动创建** | 通过 Issue 直接生成对应分支，自动设置 `origin/<branch>`，减少手动操作。 |
| **多仓库工作区** | 在同一界面管理多个仓库，支持快速切换与批量操作。 |
| **Git Flow / GitHub Flow 集成** | 内置常用工作流模板，自动生成分支命名、合并策略。 |
| **与编辑器集成** | 可直接在配置的编辑器中打开文件，支持 VS Code、Sublime 等。 |
| **冲突解决助手** | 提供可视化冲突标记与快速解决方案。 |
| **自定义脚本** | 支持在合并前/后执行自定义脚本，满足 CI/CD 需求。 |
| **安全与权限** | 支持 GitHub PAT、SSH key 等多种认证方式，安全性高。 |

## 功能细节  

1. **仓库克隆与初始化**  
   - 通过 URL 或本地路径快速克隆仓库。  
   - 自动检测远程分支，生成本地跟踪分支。  

2. **分支操作**  
   - 新建分支：基于当前 HEAD、特定提交或 Issue。  
   - 切换分支：双击分支列表即可切换。  
   - 合并分支：支持快进、三方合并，支持冲突可视化。  
   - 删除分支：本地/远程分支一键删除。  

3. **Pull Request 管理**  
   - 查看 PR 列表、状态、评论。  
   - 创建 PR：选择 base 与 head 分支，填写标题/描述。  
   - 合并 PR：一键合并，支持 `squash`、`rebase`、`merge` 三种策略。  
   - 关闭 PR：直接在客户端关闭。  

4. **Issue 与分支映射**  
   - 通过 Issue ID 自动生成分支名（如 `feature/<issue-id>-<title>`）。  
   - 自动推送分支到远程。  

5. **多仓库与工作区**  
   - 在同一窗口管理多个仓库，支持标签式切换。  
   - 统一设置 Git 配置（用户名、邮箱、SSH key）。  

6. **编辑器集成**  
   - 配置默认编辑器，双击文件即可在编辑器中打开。  
   - 支持插件系统，扩展功能。  

7. **脚本与自动化**  
   - 合并前/后执行自定义脚本（如单元测试、代码检查）。  
   - 通过 `gitbutler run` 命令执行脚本。

## 用法示例  

1. **安装**  
   - Windows/macOS：下载对应安装包，双击安装。  
   - Linux：使用 Snap/Flatpak 或手动安装。  

2. **首次启动**  
   - 打开应用，选择 `Add Repository`，输入 Git URL 或本地路径。  
   - 选择认证方式（PAT 或 SSH key）。  

3. **创建分支**  
   - 右键仓库 → `New Branch` → 选择 `From Issue`，输入 Issue ID。  

4. **提交与推送**  
   - 在 `Changes` 面板中选择文件 → 输入 Commit Message → `Commit`。  
   - 点击 `Push` 推送到远程。  

5. **创建 Pull Request**  
   - 在 `Pull Requests` 选项卡 → `Create PR` → 选择 base 与 head → 填写信息 → `Create`。  

6. **合并 PR**  
   - 在 PR 列表中选择 PR → 选择合并策略 → `Merge`。  

7. **冲突解决**  
   - 当冲突出现时，GitButler 会以可视化方式标记冲突区域。  
   - 直接在编辑器中修改文件 → 保存 → `Mark Resolved` → `Commit`。
